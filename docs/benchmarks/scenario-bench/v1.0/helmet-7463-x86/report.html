Evaluation Criteria

Capacity ConclusionScenarioBench scans channel counts sequentially, so the capacity can be reported directly. The capacity limit is the last channel count that completed and passed the report thresholds.
Channel-count PASS/FAILEach task selects its evaluation strategy by task type. CV uses critical-path latency, detection-node latency, and discard rate by default. VLM uses analysis-FPS compliance and sampling-miss rate by default, with optional end-to-end latency checks. The global average discard-rate threshold is 0.05.
Bottleneck StopRuntime protection stops the run before excessive load continues. CV uses stable-window FPS falling below half of the baseline as a guard. VLM uses target-FPS compliance so low-FPS tasks are not misclassified by short windows. Discard rate, CPU, memory, and other guards still apply.
Sampling WindowSamples are collected about every 3 seconds. Step summaries use the second half of samples in each step as the stable window.
Failure ReasonFailure reasons in the tables come from failed threshold checks. If bottleneck protection also stopped the run, the top banner shows the runtime reason that triggered early stop.
