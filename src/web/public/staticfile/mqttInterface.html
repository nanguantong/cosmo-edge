AI Box MQTT 协议接入

适用于盒子通过 MQTT 与平台进行注册、心跳和业务透传交互。

连接认证

模式字段说明
默认 IoT 认证clientId设备 SN
默认 IoT 认证usernameaibox::{设备SN}
默认 IoT 认证password由 MQTT 客户端按 IoT 认证参数生成/校验。
普通账号密码clientId、username、password使用页面或配置中设置的值。

Topic

方向topic说明
平台到设备/p2d/aibox/{设备SN}设备订阅，接收业务请求和注册响应。
设备到平台/d2p/aibox平台注册，接收设备注册和业务响应。
平台到设备/p2d/aibox/heartbeat/{设备SN}设备订阅，接收心跳响应。
设备到平台/d2p/aibox/heartbeat平台订阅，接收设备心跳。

通用消息头

字段类型说明
requestIdstring请求 ID，用于响应关联。
actionstring业务动作。注册和心跳可为空；业务透传必须是完整接口路径。
deviceSnstring设备 SN。
msgTypestringregister、heartbeat、request、response。

整体流程

设备连接 MQTT Broker 后，先订阅 /p2d/aibox/{设备SN} 和 /p2d/aibox/heartbeat/{设备SN}，然后向 /d2p/aibox 发送注册消息。注册成功后开始心跳，每 30 秒向 /d2p/aibox/heartbeat 上报一次。注册失败时当前实现每 10 秒重试。

设备注册

设备到平台

topic：/d2p/aibox

{
  "head": {"requestId":"b5e5035f-9b1e-4578-b3ac-1bed11272c49","action":"","deviceSn":"ai-123123123123","msgType":"response"},
  "body": {"devId":"ai-123123123123","supplier":"CWAI","aiHostVersion":"1.0.0","engineType":"CWNN","deviceModel":"CWAI-AIBOX","devType":2}
}
body 字段类型是否必须示例说明
devIdstring是ai-123123123123设备 ID，通常与设备 SN 一致。
supplierstring否CWAI供应商标识。
aiHostVersionstring否1.0.0AI 核心版本号。
engineTypestring否CWNN算法引擎类型。
deviceModelstring否CWAI-AIBOX设备型号。
devTypenumber否2设备类型，按平台枚举填写。

平台到设备

topic：/p2d/aibox/{设备SN}

{
  "head": {"requestId":"b5e5035f-9b1e-4578-b3ac-1bed11272c49","action":"","deviceSn":"ai-123123123123","msgType":"register"},
  "body": {"resCode":1,"resMsg":[],"resData":{}}
}
body 字段类型是否必须示例说明
resCodenumber是1注册结果：1 成功，0 失败。
resMsgobject[]否[]返回消息列表。
resDataobject否{}返回数据，注册响应通常为空对象。

业务透传

平台下发业务请求时，设备只处理 head.deviceSn 与本机一致且 head.msgType 为 request 的消息。设备处理后向 /d2p/aibox 返回 response。body 是字符串，内部为业务请求或响应 JSON。

平台到设备请求

topic：/p2d/aibox/{设备SN}

{
  "head": {"requestId":"3ee38537-e0ba-42b6-b909-feb7d6bfcaf3","action":"/gtw/cwai/System/QueryDeviceInfo","deviceSn":"ai-123123123123","msgType":"request"},
  "body": "{}"
}

设备到平台响应

topic：/d2p/aibox

{
  "head": {"requestId":"3ee38537-e0ba-42b6-b909-feb7d6bfcaf3","action":"/gtw/cwai/System/QueryDeviceInfo","deviceSn":"ai-123123123123","msgType":"response"},
  "body": "{\"resCode\":1,\"resMsg\":[{\"msgCode\":\"0\",\"msgText\":\"操作成功\"}]}"
}

常用业务接口说明

本节中的"发送消息"展示的是 MQTT 外层 body 字符串内部的业务 JSON。实际 MQTT 下发时，请将该业务 JSON 执行 JSON.stringify 后放入外层 body 字段。

通道相关接口当前使用 channelName、url、videoChannelId。旧字段 cameraName、rtspUrl、cameraId 不会被当前 DTO 正确接收。

Camera/Add 成功响应中的 resData.id 就是通道 ID，后续删除、更新、抓图和任务配置接口分别填入 videoChannelId 或 channelId。

B01-通道创建

Path/gtw/cwai/Camera/Add
MethodPOST
接口描述创建实时流、点播、ONVIF、本地视频或 USB 通道。实时 RTSP 接入时必须填写 channelName、url、channelType。

发送消息

{
  "channelType": 0,
  "channelName": "测试实时通道",
  "url": "rtsp://:@:554/h264/ch1/main/av_stream",
  "channelCode": "",
  "channelPic": ""
}

完整 MQTT 下发示例

{
  "head": {
    "requestId": "7b1d570f-eb9a-4668-91ff-cd04b939eab9",
    "action": "/gtw/cwai/Camera/Add",
    "deviceSn": "",
    "msgType": "request"
  },
  "body": "{\"channelType\":0,\"channelName\":\"测试实时通道\",\"url\":\"rtsp://:@:554/h264/ch1/main/av_stream\",\"channelCode\":\"\",\"channelPic\":\"\"}"
}

响应消息

{
  "resCode": 1,
  "resMsg": [{"msgCode": "0", "msgText": "操作成功"}],
  "resData": {"id": "RT0000000002"}
}
请求字段类型是否必须示例备注
channelTypenumber是0通道类型：0 直播通道；1 点播通道；2 ONVIF；3 本地视频；6 USB。
channelNamestring建议必填测试实时通道通道名称，页面展示用。
urlstring实时流必填rtsp://...视频流地址。旧字段 rtspUrl 无效。
channelCodestring否CAM-001外部通道号，可为空。
channelPicstring否/data/pic/1.jpg通道封面或抓图路径，可为空。
返回字段类型说明
resCodenumber1 成功，0 失败。
resMsg[].msgCodestring业务消息编码。
resMsg[].msgTextstring业务消息文本。
resData.idstring新增通道 ID，例如 RT0000000002。后续接口使用该值。

{
  "confVersionId": "CONF0000000001",

  "algorithmId": "ALG0000000001",
  "algorithmCategory": "1",
  "algorithmUsage": "0",
  "remark": "编排测试",

  "algorithmProcessdata": "{}",
  "algorithmMetadata": "{}",

}

响应消息

{
  "resCode": 1,
  "resMsg": [{"msgCode": "0", "msgText": "操作成功"}]
}
请求字段类型是否必须示例备注
confVersionIdstring是CONF0000000001编排配置版本 ID。
configVersionNamestring否安全帽编排-v1编排配置版本名称。
algorithmIdstring是ALG0000000001算法 ID。
algorithmCategorystring否1算法分类。
algorithmUsagestring否0算法用途。
remarkstring否编排测试备注。
atomicListstring否[]原子能力列表，当前 DTO 为字符串，传 JSON 字符串。
algorithmProcessdatastring否{}算法流程数据，当前 DTO 为字符串，传 JSON 字符串。
algorithmMetadatastring否{}算法元数据，当前 DTO 为字符串，传 JSON 字符串。
filePathstring否/data/algorithm/layout/helmet.json编排文件路径。
